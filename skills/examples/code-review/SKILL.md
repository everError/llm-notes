# Code Review Skill

## Description

코드 리뷰를 요청할 때 사용합니다.
트리거: '코드 리뷰', '리뷰해줘', 'PR 검토', '코드 검토', '이 코드 봐줘'
Do NOT use for: 코드 작성, 버그 수정, 리팩토링 요청

## Instructions

1. 리뷰 대상 코드가 제공되었는지 확인합니다. 없으면 사용자에게 요청합니다.
2. 언어를 특정할 수 없으면 사용자에게 확인합니다.
3. review.py를 실행하여 정적 분석 결과를 먼저 수집합니다.
4. 정적 분석 결과와 직접 검토를 종합하여 아래 관점 순서로 리뷰합니다.
   - 버그/논리 오류
   - 보안 취약점
   - 성능 이슈
   - 가독성/네이밍
   - 중복 코드
5. 각 이슈는 Critical / Warning / Suggestion 등급으로 분류합니다.
6. template.md 형식에 맞춰 결과를 출력합니다.

## Constraints

- 스타일 취향(탭 vs 스페이스 등)은 지적하지 않습니다
- 동작하는 코드에 대해 전면 재작성을 권하지 않습니다
- 이슈가 없으면 "이슈 없음"으로 명시하고 잘된 점을 간단히 언급합니다

## Files

- `review.py` — 정적 분석 실행 스크립트
- `template.md` — 리뷰 결과 출력 템플릿
