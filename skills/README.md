# Skills (프롬프트 스킬 시스템)

LLM이 특정 작업을 수행할 때, 미리 정의된 스킬 파일을 참조하여 최적의 결과를 내도록 하는 시스템입니다.

---

## 개념

기존 프롬프트 엔지니어링은 매번 사용자가 상세한 지시를 내려야 했습니다.
스킬 시스템은 이를 **재사용 가능한 단위로 모듈화**하는 접근입니다.

- 도메인별 전문 프롬프트를 `SKILL.md` 파일로 정의
- LLM이 사용자 요청을 분석하여 적합한 스킬을 자동 선택
- 스킬 내부의 베스트 프랙티스에 따라 작업 수행

단순히 "잘 써달라"가 아니라, **검증된 절차와 규칙을 따르게 하는 것**이 핵심입니다.

---

## 스킬 구조

하나의 스킬은 보통 다음 요소로 구성됩니다.

```
skills/
├── docx/
│   └── SKILL.md
├── pdf/
│   └── SKILL.md
├── xlsx/
│   └── SKILL.md
└── custom-skill/
    ├── SKILL.md
    └── (템플릿, 예시 파일 등)
```

### SKILL.md 구성 요소

| 항목            | 설명                                                        |
| --------------- | ----------------------------------------------------------- |
| **name**        | 스킬 식별자                                                 |
| **description** | 스킬이 언제 트리거되어야 하는지 설명 (LLM이 이걸 보고 선택) |
| **지시사항**    | 실제 작업 수행 시 따라야 할 절차, 규칙, 제약조건            |
| **예시/템플릿** | 참고할 수 있는 출력 예시나 파일 템플릿                      |

---

## 동작 흐름

```
사용자 요청
    ↓
LLM이 등록된 스킬 목록(description)을 스캔
    ↓
요청에 맞는 스킬 선택
    ↓
SKILL.md를 읽어 지시사항 로드
    ↓
지시사항에 따라 작업 수행
```

핵심은 **description이 트리거 역할**을 한다는 점입니다. description이 부정확하면 스킬이 선택되지 않거나 엉뚱한 스킬이 선택됩니다.

---

## 스킬 작성 가이드

### 좋은 description 작성법

description은 LLM이 스킬을 선택할지 판단하는 유일한 기준입니다.

**포함해야 할 것**

- 이 스킬이 처리하는 작업 유형
- 트리거 키워드 (사용자가 쓸 법한 표현)
- 이 스킬을 쓰지 말아야 하는 경우

**예시**

```
Use this skill when the user wants to create, edit, or manipulate Word documents (.docx).
Triggers: 'Word doc', '.docx', 'report', 'memo', 'letter' 등
Do NOT use for PDFs, spreadsheets, or Google Docs.
```

트리거 조건과 제외 조건을 모두 명시하는 것이 오작동을 줄이는 핵심입니다.

### 지시사항 작성법

- 작업 순서를 단계별로 명시
- 흔한 실수나 함정을 미리 경고
- 출력 형식과 품질 기준을 구체적으로 정의
- 필요 시 템플릿 파일 경로 포함

---

## 커스텀 스킬 예시

아래는 API 문서 생성 스킬의 간단한 예시입니다.

```markdown
# API Documentation Skill

## Description

API 엔드포인트 명세서를 작성할 때 사용합니다.
트리거: 'API 문서', 'endpoint 정리', 'swagger', 'API spec'

## Instructions

1. 엔드포인트를 HTTP 메서드별로 그룹핑
2. 각 엔드포인트에 경로, 메서드, 설명, 요청/응답 예시 포함
3. 인증 방식은 문서 상단에 별도 섹션으로 작성
4. 에러 코드는 공통 테이블로 정리
```

---

## 스킬 시스템의 장점

- **일관성**: 같은 유형의 작업은 항상 같은 품질 기준으로 수행
- **재사용**: 한 번 검증한 프롬프트를 반복 활용
- **협업**: 팀원 간 프롬프트 노하우를 공유 가능
- **확장성**: 새로운 도메인은 스킬 파일만 추가하면 됨
