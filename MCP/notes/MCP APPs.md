## MCP Apps

### 한 줄 요약

> MCP 서버가 텍스트·JSON 데이터만 반환하던 한계를 넘어, **대화창 안에 직접 인터랙티브 UI를 렌더링**할 수 있게 해주는 MCP 공식 확장 스펙입니다.

---

### 배경

기존 MCP는 도구(Tool) 실행 결과를 텍스트 또는 구조화된 JSON으로만 반환할 수 있었습니다. 예를 들어 데이터 시각화 서버가 차트 데이터를 JSON으로 넘기면, 그걸 어떻게 그릴지는 클라이언트(호스트 앱)가 직접 구현해야 했습니다. 이 문제를 해결하기 위해 커뮤니티마다 제각각 방식으로 UI를 구현했고, 표준 없는 다양한 구현이 생겨 생태계 파편화 위험이 커졌습니다.

이에 MCP Apps가 공식 확장으로 정식 출시되었고, 도구가 대화창에서 직접 렌더링되는 대시보드, 폼, 시각화, 멀티스텝 워크플로우 같은 인터랙티브 UI 컴포넌트를 반환할 수 있게 되었습니다.

Anthropic과 OpenAI 메인테이너, 그리고 커뮤니티 프로젝트인 MCP-UI가 공동으로 설계했습니다.

---

### 핵심 동작 방식

**UI 리소스 사전 선언**

```javascript
// 서버가 UI 리소스를 등록
{
  uri: "ui://charts/bar-chart",
  name: "Bar Chart Viewer",
  mimeType: "text/html+mcp"
}

// 도구(Tool)가 해당 UI 리소스를 메타데이터로 참조
{
  name: "visualize_data_as_bar_chart",
  _meta: {
    "ui/resourceUri": "ui://charts/bar-chart"
  }
}
```

도구 실행 전에 호스트가 UI 템플릿을 미리 가져와 검토할 수 있어 성능과 보안이 향상됩니다. 정적인 UI 템플릿과 동적인 도구 실행 결과를 분리해서 캐싱도 효율적으로 할 수 있습니다.

**통신 방식**

별도 프로토콜을 새로 만들지 않고, 기존 MCP의 **JSON-RPC over postMessage**를 그대로 사용합니다. UI와 호스트 간 모든 통신이 구조화되고 감사(audit) 가능합니다.

**렌더링**

현재는 `text/html` 콘텐츠만 지원하며, 샌드박스 처리된 `<iframe>` 안에서 렌더링됩니다.

---

### 보안 설계

| 레이어             | 내용                                               |
| ------------------ | -------------------------------------------------- |
| iframe 샌드박싱    | UI 콘텐츠는 제한된 권한의 샌드박스 내에서만 실행   |
| 사전 선언 템플릿   | 호스트가 렌더링 전 HTML 검토 가능                  |
| 감사 가능한 메시지 | UI↔호스트 통신이 JSON-RPC로 로깅됨                 |
| 사용자 동의        | 호스트가 UI 발신 도구 호출에 명시적 승인 요구 가능 |

---

### 하위 호환성

MCP Apps는 선택적 확장입니다. 기존 구현은 변경 없이 계속 동작하고, 호스트는 자신의 속도에 맞춰 점진적으로 UI 지원을 추가할 수 있습니다. UI를 지원하지 않는 클라이언트를 위해 서버는 항상 텍스트 폴백을 함께 제공해야 합니다.

---

### 기존 방식과 비교

```
기존 MCP
  도구 실행 → JSON 반환 → 클라이언트가 직접 UI 렌더링 구현

MCP Apps
  도구 실행 → UI 리소스(HTML) + 데이터 반환 → 대화창에 바로 렌더링
```

---

### 현재 상태 (2025년 11월 기준)

공식 확장 스펙(SEP-1865)으로 제안되었고 얼리 액세스 SDK가 공개된 상태입니다. Postman, Shopify, Hugging Face, ElevenLabs 등 MCP-UI 기반으로 이미 도입한 사례가 있습니다. 현재는 HTML만 지원하며, 외부 URL, 네이티브 위젯 등은 이후 버전에서 추가될 예정입니다.
